# Снятие каталога вызовов

Цель — собрать `actions.draft.json` (черновик с реальными `action`/`endpoint` твоего портала),
из которого затем делается `actions.json`. Два способа: **A — авто-запись расширением** (проще,
без DevTools) и **B — HAR вручную**.

---

## Вариант A — авто-запись расширением (рекомендуется)

Расширение в capture-сборке перехватывает все API-вызовы страницы и стримит их локальному
рекордеру, который пишет `actions.draft.json` и показывает покрытие Триады в реальном времени.

1. Собери **capture**-сборку расширения и перезагрузи его:
   ```bash
   bun run build:ext:capture
   ```
   `chrome://extensions` → у расширения нажми «Обновить» (или Load unpacked → `extension/dist/`).
2. Запусти рекордер (использует тот же `.env`):
   ```bash
   bun run capture
   ```
   Он слушает мост и пишет `actions.draft.json`. Оставь его запущенным.
3. Открой вкладку портала — внизу справа появится красный бейдж **● REC**. Прокликай по каждому
   домену Триады: задачи (список + одна задача), проекты (список групп + внутрь группы),
   чаты (мессенджер + диалог + прокрутка истории).
4. Следи за терминалом рекордера: строки `+ ajax POST tasks.task.list` — новые пойманные вызовы;
   `still missing: projects` — домен ещё не покрыт. Когда увидишь `✓ all triad domains captured` —
   нажми **Ctrl-C**. `actions.draft.json` готов.
5. **Верни обычную сборку** (важно): capture-сборка вешает хук на `fetch` и рисует бейдж —
   для повседневной работы агента это не нужно.
   ```bash
   bun run build:ext
   ```
   И снова «Обновить» расширение.

Дальше → раздел «Собрать `actions.json`» ниже.

---

## Вариант B — HAR вручную

1. Открой вкладку своего Bitrix24, залогинься.
2. DevTools → Network → включи "Preserve log" (фильтр НЕ ставь — ловим всё).
3. Прокликай те же действия по Триаде (см. пункт A.3).
4. ПКМ по списку → "Save all as HAR with content" → `captured.har` (в корень проекта).
5. `bun run catalog:draft` → создаст `actions.draft.json` в корне (скрипт всегда работает из корня проекта).
   Для другого имени/пути HAR: `bun run catalog <путь.har> > actions.draft.json`.
6. Прочитай предупреждения в stderr: если по какому-то домену "nothing captured" —
   вернись в браузер и прокликай его ещё раз.

---

## Собрать `actions.json`

Из `actions.draft.json` выбери нужные вызовы и присвой понятные ключи, которые ждут инструменты:
`tasks.list`, `task.get`, `projects.list`, `chats.recent`, `chat.messages`
(можно меньше — недостающие инструменты просто не зарегистрируются). Поля
`endpoint`/`action`/`method` оставь как в черновике; `transport`/`sampleParams` можно не переносить.

Мутирующие вызовы (`…add`/`…update`/`…delete`/…) допустимы — загрузчик каталога их больше
не отклоняет; allowlist (сам факт присутствия записи в `actions.json`) — единственная граница.

Примечание: HAR-парсер (вариант B) читает параметры из тела POST, а не из query-строки URL —
GET-вызовы с params/sessid в query захватятся с пустыми params (для POST-based Триады не проблема).
Авто-запись (вариант A) берёт тело fetch/XHR и отсекает статику по расширению файла.
